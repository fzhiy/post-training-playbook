2. 直觉与复杂度 / Intuition & Complexity

无 Cache vs 有 Cache 解码对比：

无 Cache 有 KV Cache
第 ttt 步 Attention 计算 O(t⋅d)O(t \cdot d)O(t⋅d) O(1⋅d)O(1 \cdot d)O(1⋅d)（仅新 token 作 query）
生成 nnn token 总量 O(n2⋅d)O(n^2 \cdot d)O(n2⋅d) O(n⋅d)O(n \cdot d)O(n⋅d)（缓存线性增长）

直觉： KV Cache 本质是用 内存换计算——将之前所有 step 的 Key/Value 向量缓存起来，避免重复计算。Prefill 一次处理全部 prompt 填充缓存，之后每步 decode 只需处理一个 token，Attention 的 query 维度恒为 1。

因果遮罩的精妙之处： 当 decode 阶段 Tcur=1T_{\text{cur}} = 1Tcur​=1 时，遮罩从 Tkv×TkvT_{\text{kv}} \times T_{\text{kv}}Tkv​×Tkv​ 矩阵中取最后 1 行，确保新 token 只能看到它自己及之前所有位置。
