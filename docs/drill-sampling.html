从头实现温度 / top-k / top-p (nucleus) 采样学习演练 README

本演练基于纯 PyTorch 从头实现温度缩放、top-k 和 top-p（nucleus）采样，用于从 logits 中采样 token。目录中仅包含三个文件，无其他依赖。

数学基础 / Mathematical Foundations

所有操作基于模型输出的 logits 向量 z∈RVz \in \mathbb{R}^{V}z∈RV，其中 VVV 为词表大小。采样过程如下：

  1. 温度缩放 / Temperature Scaling
    给定温度 τ>0\tau > 0τ>0，缩放后的 logits 为 zi/τz_i / \tauzi​/τ，然后通过 softmax 转换为概率：
    pi=ezi/τ∑j=1Vezj/τp_i = \frac{e^{z_i / \tau}}{\sum_{j=1}^V e^{z_j / \tau}}pi​=∑j=1V​ezj​/τezi​/τ​

    • τ→0\tau \to 0τ→0：分布趋近贪心解码（argmax）。
    • τ→∞\tau \to \inftyτ→∞：分布趋近均匀分布。
    • τ=1.0\tau = 1.0τ=1.0：无变化。

  2. Top-k 过滤 / Top-k Filtering
    对每个样本，保留 logits 中最大的 kkk 个值，其余设为 −∞-\infty−∞：
    z~i={zi,if zi∈top-k values−∞,otherwise\tilde{z}_i = \begin{cases} z_i, & \text{if } z_i \in \text{top-}k \text{ values} \\ -\infty, & \text{otherwise} \end{cases}z~i​={zi​,−∞,​if zi​∈top-k valuesotherwise​
    等价于 mask 掉非 top-k 的 token。

  3. Top-p（Nucleus）过滤 / Top-p Filtering
    先将 logits 转换为概率 pi=softmax(zi)p_i = \text{softmax}(z_i)pi​=softmax(zi​)，并按降序排列得到 p(1)≥p(2)≥⋯≥p(V)p_{(1)} \geq p_{(2)} \geq \dots \geq p_{(V)}p(1)​≥p(2)​≥⋯≥p(V)​。计算累积概率：
    cumulativem=∑j=1mp(j)\text{cumulative}_m = \sum_{j=1}^m p_{(j)}cumulativem​=j=1∑m​p(j)​
    找到最小的 mmm 使得 cumulativem≥p\text{cumulative}_m \geq pcumulativem​≥p（ppp 为 top_p 阈值），则保留前 mmm 个 token（至少保留 min_tokens_to_keep 个），其余设为 −∞-\infty−∞。在代码中，通过移位累积和实现 mask。

  4. 采样 / Sampling
    应用上述过滤后，对 filtered logits 计算 softmax 概率，并从多项分布中采样一个 token：
    next_token∼Multinomial(softmax(z~))\text{next\_token} \sim \text{Multinomial}(\text{softmax}(\tilde{z}))next_token∼Multinomial(softmax(z~))
    处理顺序为：温度缩放 → top-k → top-p → 多项采样。

直觉与复杂度 / Intuition and Complexity

  • 直觉 / Intuition：

    • 温度控制随机性：低温使输出更确定（偏向高概率 token），高温增加多样性。
    • Top-k 直接限制选择范围到最可能的 kkk 个 token，简单但固定。
    • Top-p (nucleus) 动态调整候选集，基于概率质量累积，确保多样性同时保持连贯性。

  • 复杂度 / Complexity：

    • 温度缩放：O(1)O(1)O(1) 逐元素操作，向量化高效。
    • Top-k：使用 torch.topk，时间复杂度约为 O(Vlog⁡k)O(V \log k)O(Vlogk)，但实践中常为线性。
    • Top-p：需排序和累积和，时间复杂度 O(Vlog⁡V)O(V \log V)O(VlogV)，但批量处理时被并行化。
      整体，对于批量大小 BBB 和词表大小 VVV，主要操作为 O(BV)O(BV)O(BV) 或 O(BVlog⁡V)O(BV \log V)O(BVlogV)，适合 GPU 加速。
